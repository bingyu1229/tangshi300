# Tangshi300 UI 设计说明

最后更新：2026-06-22

## 设计方向

Tangshi300 应该像一张安静的现代书桌铺在宣纸上：温润纸色、克制水墨景致、朱砂印章和主操作、青玉蓝搜索控件，以及易读的中文衬线字体。

界面始终以任务为先。装饰用于营造气氛，但不能降低扫读速度、遮挡控件或削弱文本对比度。

## 字体

沿用 `ui/visual-inkWash-preview.html` 的字体方向：

- 主要可读中文文本：`Noto Serif SC`，回退到 `Songti SC`、`SimSun`、serif。
- 诗意强调和诗题：`Ma Shan Zheng`，回退到 `STKaiti`、`KaiTi`、cursive。
- 密集 UI 文本应保持易读和安静。
- 毛笔/书法字体只用于标题和强调时刻，不用于工具型标签。

## 导航

生产导航包含四项：

- 首页：`/`
- 诗库：`/search`
- 学习：`/learn`
- 复习册：`/review-book`

桌面头部应保持紧凑，并围绕应用主功能区居中组织。移动端使用同样四项作为底部标签。

已移除：

- 偏个人资料的导航。
- 头像/下拉入口。

## 页面指导

### 首页

- 以当前水墨首页作为基准。
- 搜索提交到 `/search?q=...`。
- 状态胶囊打开复习册。
- 推荐诗卡片打开诗词详情。
- 继续学习打开今日测试。
- 最近学习使用真实进度数据。
- 推荐列表在客户端轮换。
- 进度环使用真实复习册进度。

### 搜索 / 诗库

- 保持青玉蓝搜索按钮和安静的结果密度。
- 来自 `/search?q=...` 的查询必须初始化输入框和结果列表。
- 筛选 chip 可交互，并会重置分页。
- 结果卡片使用一致的缩略图尺寸、可读摘录和清晰详情入口。

### 诗词详情

- 书法/Kaiti 标题、衬线元数据和 Kaiti 诗句应保持排版焦点。
- 插图起辅助作用，不应喧宾夺主。
- 音频兜底控件应保持在诗词头部附近可见。
- 注释、译文和赏析保持紧凑、易读。

### 复习册

- 数字和进度由真实数据驱动。
- 移除动作要清晰，但视觉上应比主要学习动作更安静。
- 空状态应引导学习者回到学习流程。

### 默写测试

- 提示句和答案输入框是视觉焦点。
- 正确和错误状态需要明确但克制的反馈。
- 提交后的下一步动作应明显。

## QA 清单

- `npm run build` 通过。
- 激活导航状态与路由匹配。
- 头部和底部标签不与内容重叠。
- 首页和搜索表单支持查询字符串。
- 进度环显示正确百分比。
- 卡片保持 8px 圆角和一致间距。
- 水墨图像不降低文本对比度。
- 按钮文本在移动端和桌面端都能容纳。
- 移动布局保持可读，不出现水平溢出。
- 诗词详情打印样式干净，并省略交互式 chrome。
