Tangshi300 Architecture

Local-first poetry learning, from XML to daily practice.

Tangshi300 is a compact system: a deterministic corpus pipeline feeds a local SQLite database, Next.js renders the study experience, and small API routes update learner state without needing a remote backend.

Next.js App Router sql.js SQLite Weighted search index Review-book state machine Audio fallback ready
1
PrepareParse XML, sample 50 poems, and generate JSON.
2
PersistSeed SQLite with poems, progress tables, recommendations, and search terms.
3

4
PracticeAPI routes handle search refreshes, test answers, and review-book removal.

Data Pipeline

Source tangshi300.xml Raw poem corpus committed with the project.
->
Convert scripts/convert-tangshi-xml.ts Parses 321 poems and writes full plus v1 sample JSON.
->
Seed scripts/seed-db.ts Creates SQLite rows and weighted CJK/Latin search terms.

Runtime Architecture

Pages

app/page.tsxDaily recommendation, progress, recent learning, and homepage search.
app/search/page.tsxServer-initialized poem library search.
app/poems/[id]Poem detail and dictation test routes.
app/review-bookMastered poems, progress, and removal flow.

Domain Layer

lib/db/poems.tsRepository for listing, search, detail, recommendations, review, and tests.
lib/db/sqlite.tsLoads sql.js, ensures schema, persists the database file.
lib/text.tsBuilds search terms and normalizes recitation answers.
lib/poems/import.tsOwns XML parsing and stable v1 sampling.

Interactions

components/SearchClient.tsxFetch search, filter chips, highlighting, and pagination.
components/TestClient.tsxAnswer capture and feedback for dictation practice.
components/AudioPlayer.tsxGenerated file playback when available, browser speech fallback otherwise.
components/ReviewBookClient.tsxReview list controls and remove/relearn actions.

SQLite Schema

poems

Canonical poem content, lines, metadata, notes, translation, appreciation, and source.

poem_audio

Optional generated audio metadata linked to files under public/audio.

learning_progress

Current learner state, mastered time, latest test time, and answer counts.

daily_recommendations

One persisted poem pick per calendar day.

poem_search_terms

Weighted terms for title, author, genre, content, notes, translation, and appreciation search.
