Tangshi300 Architecture

本地优先的诗词学习，从 XML 走到每日练习。

Tangshi300 是一个紧凑的系统：确定性语料管线写入本地 SQLite 数据库， Next.js 渲染学习体验，小型 API 路由负责更新学习者状态，无需远程后端。

Next.js App Router sql.js SQLite 加权搜索索引 复习册状态机 音频兜底就绪
1
准备解析 XML，稳定抽样 50 首诗，并生成 JSON。
2
持久化将诗词、进度表、推荐和搜索词写入 SQLite。
3
渲染服务端页面从仓储层读取数据，并水合聚焦的客户端交互。
4
练习API 路由处理搜索刷新、测试答案和复习册移除。

数据管线

Source tangshi300.xml 随项目提交的原始诗词语料。
->
Convert scripts/convert-tangshi-xml.ts 解析 321 首诗，并写出完整 JSON 和 v1 样本 JSON。
->
Seed scripts/seed-db.ts 创建 SQLite 数据行和加权 CJK/Latin 搜索词。

运行时架构

页面层

app/page.tsx每日推荐、进度、最近学习和首页搜索。
app/search/page.tsx服务端初始化的诗库搜索。
app/poems/[id]诗词详情和默写测试路由。
app/review-book已掌握诗词、进度和移除流程。

领域层

lib/db/poems.ts列表、搜索、详情、推荐、复习和测试仓储。
lib/db/sqlite.ts加载 sql.js、确保 schema、持久化数据库文件。
lib/text.ts构建搜索词并归一化默写答案。
lib/poems/import.ts负责 XML 解析和稳定 v1 采样。

交互层

components/SearchClient.tsx搜索请求、筛选 chip、高亮和分页。
components/TestClient.tsx默写练习中的答案捕获和反馈。
components/AudioPlayer.tsx优先播放生成文件，否则使用浏览器语音兜底。
components/ReviewBookClient.tsx复习列表控制和移除/重学动作。

SQLite Schema

poems

标准诗词内容、诗句、元数据、注释、译文、赏析和来源。

poem_audio

可选生成音频元数据，关联 public/audio 下的文件。

learning_progress

当前学习状态、掌握时间、最近测试时间和答题计数。

daily_recommendations

按自然日持久化的一首推荐诗。

poem_search_terms

面向标题、作者、题材、正文、注释、译文和赏析的加权搜索词。
